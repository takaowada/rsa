田村らの修正手法もKhayatの修正手法と同様に、RSA暗号をベースにしたモジュラーエキスポーネントを用いています。この手法では、誰もが知っている2つの大きな素数Q_1、Q_2を考える。また、この技術は、(1)設定、(2)暗号化、(3)復号、(4)検証の4段階から構成される。それぞれの役割は以下の通りである。
設定: 参加者C_jは暗号鍵と復号鍵のペア｛L_j, Z_j｝を持つが、この修正技法では自由裁量である。各権威者 A_i はまた、2組の暗号化・復号化鍵ペア：モジュロ Q_1 演算による暗号化のための {K_i, F_i} と、モジュロ Q_2 演算による復号化のための {H_i, G_i} を持つ。Khayatの手法と同様に、鍵ペアは A_i の秘密とし、複数の機関が同じモジュロ演算を共有する環境において、各 A_i が自分の鍵ペアを安全に使用できるようにする。H_i と K_i は、他の権威の暗号鍵を尊重した乗算形式 (H^*とK^*) でのみ開示される。すなわち、H^* = PI H_i{i=1,P}, K^* = PI K_i {i=1,P} (Pは権威の総数)である。鍵 K_i または H_i が公開されている場合、K_iF_i (mod (Q_1-1)) = K_jF_j(mod (Q_1-1)) または H_iG_i (mod (Q2-1)) = H_jG_j (mod (Q_2-1)) の関係から A_i の復号鍵 F_i は誰でも容易に計算することが可能である。ここで、任意の2つの整数 u と w に対して、u^{KiFi} (mod Q_1) = u (mod Q_1), w^{H_iG_i} (mod Q_2) = w (modQ_2), w^{L_jZ_j} (mod Q_2) = w (mod Q_2)の関係式を満たします。
暗号化: このフェーズでは、参加者 C_j は秘密の乱数 r_j を生成し、E({K^*, H^*}, {m_j, r_j}) = {E(K^*, m_jr_j) = (m_jr_j)^K^*(mod Q_1) and E(H^*, r_j) = (r_j)^H^*(mod Q2)} すなわち暗号化はデータ部分 E(K_i, m_jr_j) およびランダム化部分 E(H_i, r_j)から計算することにより彼女のメッセージ m_j を暗号化する。ここで、 r_j は C_j の秘密であり、r_jL_j から r_j を計算するのは離散対数の問題であるため、当局は m_jr_j と r_j^{L_j} から m_j を計算することはできない。
復号化: この段階では、暗号文 E({K^*, H^*}, {m_j, r_j}) は、当局 {A_1, ---, A_P} によって、E(K^*, m_jr_j)(F_1, ---． F_P) = (m_jr_j)^{K^*(F_1,---,F_P)} (mod Q_1) = m_jr_j および E(H^*, r_j)^(G_1,---,G_P) = r_jH^*^(G_1,---,G_P) (mod Q_2) = r_j を、復号化キー F_1,---,F_P と G_1,---,G_P を用いて計算することにより、当局によって復号化され、 (m_jr_j, r_j)となった。そして最後に r_j は m_jr_j からmを取り出すのに使われる。
検証: {A_1,---,A_P}の再暗号化が正しいかどうかを確認するために、Cjは E(K^*, (m_jr_j)U_j) と E(H^*, r_j^V_j) を復号するように当局に求める。したがって、各 A_i の K_i と H_i は秘密であるが、C_j は操作の正しさを確認することができる。しかし、この段階は C_j にとってオプションである。この再暗号化技術は、確率的かつ可換的であることが明らかである。
